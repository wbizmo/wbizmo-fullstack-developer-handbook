# Lesson 04: Development Tools and Installation

## Learning Objectives

By the end of this lesson, you should be able to:

- Understand what developer tools are.
- Know which tools are essential for beginners.
- Install software safely.
- Identify official download sources.
- Understand the role of VS Code, Git, Node.js, browsers, and terminals.
- Prepare a computer for software development.

## What Are Development Tools?

Development tools are software applications that help developers write, test, debug, manage, and deploy code.

Without development tools, creating software would be extremely difficult.

Think of development tools as the equipment a carpenter uses to build furniture.

Developers use tools to build software.

## Essential Tools for Beginners

A beginner web developer should install:

- A web browser
- Visual Studio Code
- Git
- Node.js
- A terminal application

These tools form the foundation of modern development.

## Tool 1: Web Browser

A browser allows you to view websites and test web applications.

Popular options:

- Google Chrome
- Microsoft Edge
- Mozilla Firefox

Developers use browsers to:

- Test websites
- Inspect HTML and CSS
- Debug JavaScript
- Monitor network requests

## Tool 2: Visual Studio Code

Visual Studio Code (VS Code) is one of the most popular code editors in the world.

Common uses:

- Writing code
- Editing files
- Running terminals
- Managing projects
- Installing extensions

Features:

- Syntax highlighting
- Integrated terminal
- Git integration
- Extensions marketplace
- Debugging support

Official website:

https://code.visualstudio.com

## Tool 3: Git

Git is a version control system.

It allows developers to:

- Track changes
- Create backups
- Collaborate
- Restore previous versions

Git is used by almost every professional software team.

Official website:

https://git-scm.com

## Tool 4: GitHub

GitHub is a platform built around Git.

Developers use GitHub to:

- Store repositories
- Share code
- Collaborate
- Manage releases
- Track issues

Official website:

https://github.com

## Tool 5: Node.js

Node.js allows JavaScript to run outside the browser.

Many modern tools require Node.js.

Examples:

- React
- Next.js
- Vite
- ESLint
- npm

Official website:

https://nodejs.org

## Tool 6: Terminal

The terminal allows developers to interact directly with the operating system.

Examples:

- PowerShell
- Command Prompt
- Git Bash
- Windows Terminal

Many development tasks are performed through the terminal.

## Installing Software Safely

Always download software from official sources.

Good example:

Download Git from:

https://git-scm.com

Bad example:

Download Git from:

random-free-download-site.example

Official sources reduce security risks.

## How Software Installation Works

Typical installation process:

1. Visit official website.
2. Download installer.
3. Run installer.
4. Follow installation wizard.
5. Verify installation.

## Verifying Installations

After installing Node.js:

    node -v

After installing npm:

    npm -v

After installing Git:

    git --version

Version numbers confirm successful installation.

## Why Developers Verify Installations

Sometimes software installs incorrectly.

Verification helps ensure:

- Tool is installed
- Tool is available in PATH
- Tool is ready for use

## Understanding PATH

PATH is an operating system setting that tells the computer where to find programs.

Example:

When you run:

    git

The operating system searches PATH to locate Git.

Without PATH configuration:

    command not found

errors may occur.

## Common Beginner Mistakes

### Mistake 1

Installing tools from unofficial websites.

### Mistake 2

Skipping installation verification.

### Mistake 3

Ignoring installation errors.

### Mistake 4

Installing multiple conflicting versions.

### Mistake 5

Not restarting the terminal after installation.
