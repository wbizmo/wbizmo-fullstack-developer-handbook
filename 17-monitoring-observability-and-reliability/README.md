# Monitoring, Observability & Reliability

## Overview

Building software is only half the job.

Operating software is the other half.

A production system that cannot be observed cannot be operated effectively.

This module teaches the principles and practices used to monitor, troubleshoot, operate and improve production systems.

## Why This Module Matters

Many developers focus on:

- APIs
- Databases
- Frameworks

Production engineers focus on:

- Reliability
- Monitoring
- Incidents
- Availability
- Performance

This module bridges that gap.

## Learning Outcomes

By the end of this module you should be able to:

- Design monitoring systems
- Collect logs
- Collect metrics
- Understand distributed tracing
- Build alerting systems
- Define SLIs
- Define SLOs
- Define SLAs
- Respond to incidents
- Operate production systems
