# Lesson 01: Introduction To Databases

## Learning Objectives

By the end of this lesson you should be able to:

- Explain what a database is.
- Explain why databases exist.
- Understand records and tables.
- Understand persistent storage.
- Explain common database use cases.

## What Is A Database?

A database is an organized system for storing information.

Examples:

- Users
- Products
- Orders
- Books
- Transactions
- Files
- Messages

Without databases, applications forget information whenever they restart.

## Why Databases Matter

Imagine:

Cashflowr stores expenses.

Without a database:

All expenses disappear after restart.

Databases provide persistence.

## Tables

Data is commonly stored in tables.

Example:

Users

| id | name | email |
|----|------|--------|
| 1 | Ada | ada@email.com |
| 2 | John | john@email.com |

## Rows

Each record is a row.

Example:

| 1 | Ada | ada@email.com |

One user.

## Columns

Columns describe attributes.

Example:

name

email

phone

created_at
