# Lesson 01 Cheatsheet: How The Internet Works

## Internet

A global network of connected networks.

## Web

A service that runs on the internet and allows people to access websites through browsers.

## Network

A group of connected devices that can communicate.

## ISP

Internet Service Provider.

Connects users to the internet.

## Router

Directs data between networks.

## IP Address

A unique address used to identify a device or server on a network.

## Public IP

An address used on the internet.

## Private IP

An address used inside a local network.

## Data

Information sent or received by computers.

Examples:

- Text
- Images
- Videos
- HTML
- CSS
- JavaScript
- JSON

## Packet

A small piece of data sent across a network.

## Why Packets Matter

Packets make data transfer:

- More efficient
- More reliable
- Easier to route
- Easier to retry when something fails

## TCP

Transmission Control Protocol.

Helps ensure data arrives reliably and in order.

## IP

Internet Protocol.

Helps route data to the correct destination.

## UDP

User Datagram Protocol.

Faster but less focused on guaranteed delivery.

## Client

A device or program that requests resources.

Examples:

- Browser
- Mobile app
- API client

## Server

A computer or program that provides resources.

Examples:

- Website server
- API server
- Database server
- File server

## Basic Request Journey

    Browser → Router → ISP → Internet → Server → Response Back

## Key Difference

Internet:

    Network infrastructure

Web:

    Websites accessed through browsers

## Developer Takeaway

Full stack applications depend on internet communication.

Frontend apps send requests.

Backend apps receive requests.

Servers send responses.

Databases and APIs may live across networks.

Understanding the internet makes debugging and deployment easier.
