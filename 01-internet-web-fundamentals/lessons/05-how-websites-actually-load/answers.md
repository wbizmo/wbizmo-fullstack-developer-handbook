# Lesson 05 Answers: How Websites Actually Load

1. The user enters a URL.
2. DNS helps find the server IP address.
3. The browser sends an HTTP or HTTPS request.
4. The server sends a response.
5. HTML provides structure.
6. CSS provides styling and layout.
7. JavaScript provides behavior and interactivity.
8. The Network tab in browser developer tools.
9. Large files, slow servers, poor network, heavy JavaScript, and unoptimized assets can slow websites down.
10. The browser reads the URL, performs DNS lookup, connects to the server, sends a request, receives files, downloads extra resources, and renders the page.

## Exercise Solution

A typical website loading journey:

    User enters URL
    Browser performs DNS lookup
    Browser connects to server
    Browser sends request
    Server sends HTML
    Browser downloads CSS, JavaScript, images, and fonts
    Browser renders page
