# Lesson 06: Pull Requests, Issues and Collaboration

## Learning Objectives

- Understand pull requests
- Understand issues
- Understand reviews
- Understand collaboration workflows

## What Is A Pull Request?

A Pull Request (PR) proposes changes.

Example:

Feature branch:

    feature-auth

Request:

    Merge feature-auth into main

## Benefits

- Review code
- Discuss changes
- Catch bugs
- Improve quality

## What Is An Issue?

Issues track:

- Bugs
- Features
- Improvements
- Tasks

Examples:

    Fix login bug
    Add dark mode
    Improve API docs

## Code Review

Reviewers check:

- Quality
- Bugs
- Security
- Maintainability

## Collaboration Workflow

Create issue

↓

Create branch

↓

Commit work

↓

Push

↓

Open PR

↓

Review

↓

Merge

## Exercises

1. Create an issue.
2. Create a branch.
3. Open a pull request.
4. Review a PR.
5. Merge a PR.

## Summary

Issues organize work.

PRs review work.

Code reviews improve quality.
