# Microservices Architecture

## Overview

Microservices are an architectural style where systems are composed of independently deployable services.

Instead of building one large application, functionality is divided into smaller services with clear responsibilities.

This module explores how large engineering organizations design, deploy and operate microservice platforms.

## Why This Module Matters

Most modern platforms use some combination of:

- APIs
- Services
- Events
- Queues
- Workers
- Databases

Understanding microservices helps engineers build scalable systems and understand platform architecture.

## Learning Outcomes

By the end of this module you should be able to:

- Explain microservices
- Define service boundaries
- Understand service communication
- Use API gateways
- Design event-driven systems
- Handle data across services
- Build resilient services
- Observe distributed applications
- Evaluate microservice trade-offs
