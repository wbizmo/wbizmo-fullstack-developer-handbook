# Distributed Systems Fundamentals

## Overview

Distributed systems are software systems made of multiple independent components that communicate over a network.

This module teaches the core ideas behind modern backend platforms, orchestration systems, monitoring systems, queues, databases, cloud infrastructure and platform engineering.

## Why This Module Matters

Most serious backend and platform systems are distributed.

Examples:

- Kubernetes
- Redis Cluster
- PostgreSQL replication
- Kafka
- service meshes
- cloud platforms
- monitoring systems
- orchestration platforms

Distributed systems introduce hard problems:

- partial failure
- network delay
- consistency
- coordination
- leader election
- consensus
- retries
- idempotency
- service discovery

## Learning Outcomes

By the end of this module you should be able to:

- Explain distributed systems
- Understand partial failure
- Understand clocks and ordering
- Explain leader election
- Explain consensus
- Understand Raft at a beginner level
- Design service discovery
- Understand distributed locks
- Understand heartbeats
- Understand gossip protocols
- Understand sagas and distributed transactions
- Reason about distributed system trade-offs
